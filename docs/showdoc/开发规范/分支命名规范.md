# 分支命名规范

## 格式

```
类型/简短描述
```

## 命名对照表

| 类型 | 格式 | 示例 | 用途 |
|------|------|------|------|
| 新功能 | `feat/功能名` | `feat/user-auth` | 开发新功能 |
| Bug 修复 | `fix/bug描述` | `fix/login-error` | 修复 Bug |
| 文档 | `docs/描述` | `docs/api-readme` | 文档更新 |
| 重构 | `refactor/描述` | `refactor/settings` | 代码重构 |
| 紧急修复 | `hotfix/描述` | `hotfix/critical-bug` | 线上紧急修复 |
| 测试 | `test/描述` | `test/auth-unit` | 测试相关 |

## 注意事项

- 使用**英文小写**，单词用 `-` 连接
- 描述尽量简短，2-3 个单词
- 功能分支从 `develop` 创建
- `hotfix` 从 `main` 创建（紧急情况）
- PR 合并后，根据分支名**自动打标签**（CI 配置）
