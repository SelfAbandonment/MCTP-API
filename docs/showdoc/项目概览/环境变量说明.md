# 环境变量说明

项目通过 `.env` 文件管理敏感配置，使用 `python-dotenv` 加载。

## 配置方式

```bash
# 从模板创建
cp .env.example .env
# 编辑 .env 填入你的配置
```

## 变量列表

| 变量名 | 必填 | 默认值 | 说明 |
|--------|------|--------|------|
| `SECRET_KEY` | ✅ | `django-insecure-change-me-in-production` | Django 密钥，**生产环境必须修改** |
| `DEBUG` | ❌ | `True` | 调试模式，生产环境设为 `False` |
| `ALLOWED_HOSTS` | ❌ | `localhost,127.0.0.1` | 允许的主机名，多个用逗号分隔 |
| `SHOWDOC_URL` | ❌ | — | ShowDoc 服务地址（同步文档用） |
| `SHOWDOC_API_KEY` | ❌ | — | ShowDoc API Key |
| `SHOWDOC_API_TOKEN` | ❌ | — | ShowDoc API Token |

## 生成 SECRET_KEY

```bash
python -c "from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())"
```

## 注意事项

- ⚠️ `.env` 文件已在 `.gitignore` 中，**不会提交到仓库**
- 每个开发者维护自己本地的 `.env`
- `.env.example` 是模板文件，**会提交到仓库**供参考
- 生产环境建议通过服务器环境变量注入，不使用 `.env` 文件
