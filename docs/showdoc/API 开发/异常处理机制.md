# 异常处理机制

项目配置了**全局异常处理器**，将所有 DRF 异常转换为统一响应格式。

## 处理方式

```python
# settings.py 中配置
REST_FRAMEWORK = {
    "EXCEPTION_HANDLER": "mctp_api.mctp_api_core.exceptions.custom_exception_handler",
}
```

## 异常类型映射

| 异常类型 | HTTP Code | message 示例 |
|----------|-----------|-------------|
| `ValidationError` | 400 | 字段级别的错误详情 |
| `NotAuthenticated` | 401 | "未提供认证信息" |
| `AuthenticationFailed` | 401 | "认证信息无效" |
| `PermissionDenied` | 403 | "没有执行该操作的权限" |
| `NotFound` | 404 | "未找到" |
| 其他 DRF 异常 | 对应状态码 | 异常 detail |
| 未捕获的异常 | 500 | "服务器内部错误" |

## 响应示例

### 参数校验失败 (400)

```json
{
    "code": 400,
    "message": {
        "username": ["该字段是必填项。"],
        "email": ["请输入有效的电子邮件地址。"]
    },
    "data": null
}
```

### 未认证 (401)

```json
{
    "code": 401,
    "message": "未提供认证信息",
    "data": null
}
```

### 服务器错误 (500)

```json
{
    "code": 500,
    "message": "服务器内部错误",
    "data": null
}
```

> DEBUG=True 时，500 错误会在 Django 控制台打印完整 Traceback，方便调试。
