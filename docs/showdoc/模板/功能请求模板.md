# 功能请求模板

在 GitHub Issues 中提交功能需求时，会自动弹出以下模板：

---

## 需要填写的内容

| 项目 | 说明 |
|------|------|
| **功能描述** | 简要描述希望添加的功能 |
| **使用场景** | 这个功能在什么场景下需要 |
| **方案设计** | 初步的实现思路（可选） |
| **备选方案** | 替代方案或临时解决办法（可选） |

## 示例

> **功能描述：** 添加用户注册 API
>
> **使用场景：** 新用户需要通过 API 注册账号
>
> **方案设计：** POST /api/register/ 接收 username、email、password，返回用户信息和 Token
